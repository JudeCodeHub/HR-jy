# Project MasterHR - Architecture Diagram Specification

This document provides a detailed, structured overview of the MasterHR system architecture. Please use this information to generate a high-level system architecture diagram.

## 1. System Overview
MasterHR is a biological-identity-based Human Resources mobile application. It replaces traditional attendance systems (passwords/ID cards) with highly secure Face Recognition and Liveness Detection, coupled with standard HR features like task management, payroll, and chat.

## 2. Core Components

### A. Frontend / Client Layer (Mobile Application)
- **Technology:** React Native (Cross-platform for iOS and Android).
- **Users/Roles:** 
  - **HR (Admin):** Manages employees, payroll, assigns tasks, monitors performance.
  - **Employee (Staff):** Marks attendance, views tasks, chats with HR.
- **Key Modules:** 
  - Authentication UI (Email/Password)
  - Geofencing Module (Polygon-based boundary validation)
  - Camera Module (Captures image for attendance)
  - Dashboards (Tasks, Payroll, Chat, Company Profile)

### B. Primary Backend (Node.js API Server)
- **Technology:** Node.js, Express.js.
- **Port:** Runs on Port 3501.
- **Responsibilities:**
  - Acts as the main API Gateway.
  - Handles Authentication and issues JSON Web Tokens (JWT) for secure communication.
  - Processes CRUD operations for Tasks, Payroll, and Chat.
  - Serves polygon geofence coordinates to validate if the employee is inside the designated office area.
  - Commits successful attendance records to the database.

### C. AI / Machine Learning Microservice (Python Engine)
- **Technology:** Python, Flask.
- **Port:** Runs on Port 5000.
- **Storage:** Uses a local Pickle file (`EncodeFile.p`) for lightning-fast memory of face encodings.
- **Responsibilities:**
  - **Face Recognition:** Uses dlib (HOG) to extract 128 key facial landmarks and compares them using Euclidean distance.
  - **Liveness Detection (Antispoofing):** Uses a Convolutional Neural Network (CNN model `1.h5`) to detect spoofing attempts (e.g., holding up a photo or screen).
  - Returns "Match/Fail" and "Live/Spoof" results to the mobile app or main server.

### D. Database Layer
- **Technology:** MongoDB.
- **Responsibilities:**
  - Flexible, NoSQL storage.
  - Collections for: Users/Employees, Attendance_records, Tasks, Chat_messages, and Payroll details.

## 3. Key Data Flows to Illustrate

Please include these vital workflows in the diagram using arrows and labels:

### Flow 1: Initialization / Employee Registration
1. HR uses Mobile App -> Sends New Employee Data + Anchor Image.
2. Node.js Server -> Saves textual data to MongoDB.
3. Node.js Server/Mobile App -> Sends Image to Python AI Service.
4. Python AI Service -> Processes image -> Saves encoding to `EncodeFile.p`.

### Flow 2: Mark Attendance (The Core A to Z Flow)
1. **GPS Check:** Mobile App checks Geofencing (validates employee GPS against the office polygon boundary).
2. **Camera Capture:** Mobile App captures selfie.
3. **AI Verification:** Mobile App sends selfie to Python AI Service.
4. **Antispoofing:** Python CNN model verifies it's a real person (Liveness Detection).
5. **Matching:** Python Face Recognition compares selfie encoding vs `EncodeFile.p` (must match logged-in User ID).
6. **Confirmation:** Python returns success to Mobile App.
7. **Database Commit:** Mobile App sends success token to Node.js Server -> Node.js creates `Attendance_record` in MongoDB.

### Flow 3: Daily Operations (Tasks, Chat, Payroll)
1. Mobile App -> Sends API requests with JWT -> Node.js Server.
2. Node.js Server -> Reads/Writes from MongoDB -> Returns data to Mobile App.

## 4. Diagram Drawing Instructions for the AI
- **Style:** Use a High-Level System Architecture or C4 Context/Container style diagram.
- **Layout:** Place the React Native Mobile App on the left (or top), the Node.js API and Python AI Service in the middle (Backend), and the MongoDB / `EncodeFile.p` on the right (or bottom).
- **Icons:** Use recognizable icons for React Native, Node.js, Python, Flask, and MongoDB.
- **Connections:** Use solid arrows for synchronous API calls (HTTP/REST) and distinct visual styles for database reads/writes.
- **Security:** Highlight the JWT Token flow between the Mobile App and Node.js server.
